## Overview

Дизайн-система TTRPG Tools — набор личных инструментов мастера для настольных ролёвок. Эстетика: тёплая бумага и чернила. Должна читаться как страница рулбука, а не как админ-панель. Утилитарна и тиха в обычном состоянии — заметна только когда ты что-то «бросаешь».

## Style Foundations

- **Визуальный стиль:** тёплая бумага, печатный текст, минимум хрома. Без градиентов, теней-стекляшек, glassmorphism.
- **Типографика:** засечный антиквенный шрифт для заголовков (Cormorant Garamond) — отсылка к старым приключенческим модулям. Без засечек (Inter) — для основного текста и UI. Моноширинный (JetBrains Mono) — для меток d20/d66, результатов броска, индексов таблиц.
- **Палитра:** охра/сиена/жжёное дерево на тёплом небелом фоне (`#FBF7EF`). Цвет используется скупо — primary только для интерактивных элементов и подсветки результата броска.
- **Темнота:** dark mode не в первой итерации, но токены спроектированы так, чтобы инвертирование шло без полной перерисовки.

## Tokens — как они оформлены в Tailwind v4

Все токены живут в `src/styles/global.css` через `@theme` директиву. Tailwind генерирует утилитарные классы: `bg-primary`, `text-text-muted`, `font-display`, `rounded-lg` и т. д.

Цвета объявлены как HSL (`hsl(...)`) — для будущей dark-темы инверсия упирается в смену переменных light → dark в `:root[data-theme="dark"]`, не в переписывание токенов.

## Colors

- **Primary (#B45309 — burnt sienna):** кнопки действия (Бросить, Перебросить), активный таб, подсветка только что выпавшего значения.
- **Secondary (#7C2D12 — dried blood):** редкие акценты, активная ссылка, ховер primary.
- **Success (#15803D):** подтверждения, валидные состояния.
- **Warning (#A16207):** «есть неразрешённый вопрос в скобках», подсказки.
- **Danger (#B91C1C):** ошибки загрузки таблиц, неконсистентные данные.
- **Surface (#FBF7EF):** фон страниц — тёплый off-white.
- **Surface-elevated (#FFFFFF):** карточки результатов, таблицы, диалоги.
- **Text (#1C1917):** основной текст.
- **Text-muted (#57534E):** второстепенный текст, подписи, мета.
- **Border (#E7E2D5):** разделители таблиц, рамки карточек.

## Typography

- **Display / H1:** системный serif-стек (`Iowan Old Style`, `Palatino Linotype`, `URW Palladio L`, `P052`, serif) 600 — название инструмента, заголовки секций рулбука.
- **H2-H3:** тот же serif-стек 600, меньший размер — подзаголовки таблиц.
- **Body:** системный sans-стек (`Inter`, `Roboto`, `Helvetica Neue`, …, sans-serif) 400 — описания, текст MDX.
- **Label / caps:** системный mono-стек (`ui-monospace`, `Cascadia Code`, `Source Code Pro`, …, monospace) 500 uppercase letter-spaced 0.08em — «D20», «ОРИЕНТИР», «ДЕТАЛЬ», номера строк в таблицах.
- **Результат броска:** serif-стек 600, увеличенный размер — выделяет выпавший текст из обычного потока.

Используются только системные шрифты (font stacks в `--font-*` токенах `global.css`) — внешние веб-шрифты не подгружаются.

## Spacing & layout

- Базовая единица — 4px. Сетка: 4/8/12/16/24/32/48/64.
- Максимальная ширина контента страницы — `48rem` (как у книги): таблицы шире не должны быть.
- Карточка результата сверху страницы — sticky или фиксированный отступ, чтобы оставаться видимой при скролле к справочным таблицам.

## Components

Описываются по мере появления. На момент первой итерации:

### Roll button

- **Anatomy:** primary color, большая (мин. touch-target 44px), иконка `d20` опциональна.
- **States:** default → hover (secondary) → active (нажат) → loading (когда подключим AI-описания — спиннер на месте текста). Focus-visible: 2px outline в primary с offset 2px.
- **Микро-взаимодействие:** при клике результат «выкатывается» — короткая (150-200ms) анимация opacity + translate. Без шумных дайс-анимаций, без звуков.

### Result card

- Surface-elevated фон, тонкая border, скругление `lg` (12px).
- Заголовок с моно-меткой («ЛОКАЦИЯ») и кнопкой «перебросить всё» справа.
- Каждая часть результата (Ориентир / Деталь) — отдельная строка с собственной кнопкой «перебросить эту часть» (иконка стрелки-цикла, появляется на hover/focus).
- Открытые вопросы из таблицы (в скобках) — отдельной строкой, цвет text-muted, шрифт italic.

### Reference table

- Полная таблица d20 с моно-индексом слева, основным текстом справа.
- Только что выпавшая строка подсвечена primary (фон с alpha 10%, левая граница в primary).
- Hover на строке — лёгкий фон, курсор default (это справка, не интерактив).

### Biome selector

- Сегментированный контрол из 4 кнопок («Деревня» / «Лес» / «Река» / «Город людей»).
- Активная — primary fill, неактивные — outline.

## Accessibility

- Контраст: WCAG 2.2 AA минимум. Text (#1C1917) на surface (#FBF7EF) — выше 14:1. Primary (#B45309) на surface — ~5:1, ок для крупного текста и кнопок.
- Focus-visible: всегда 2px outline в primary с offset 2px. Никаких `outline: none` без замены.
- Touch-target: минимум 44×44px для всех кнопок (включая «перебросить эту часть»).
- Клавиатура: Tab проходит селектор биома → кнопку броска → каждую кнопку «перебросить часть». Enter/Space активируют. Никаких only-mouse взаимодействий.
- Цвет не единственный носитель смысла: подсвеченная строка таблицы имеет ещё и левую границу + bold.
- Шрифт результата ≥ 1.5rem, чтобы читался с расстояния «через стол».

## Writing tone

- Сжато, по-делу, без восклицаний. Тон рулбука, не маркетинга.
- Только русский. Никаких «Roll!», «Generate». Кнопки: «Бросить», «Перебросить», «Сбросить».
- Сохраняем шероховатость оригинала: вопросы в скобках после результата не сглаживаем («Что было призвано?», не «Что было призвано здесь, в этом древнем храме?»).

## Rules: Do

- Использовать семантические токены (`bg-primary`, `text-text-muted`), не сырые hex.
- Делать interaction states явными (default/hover/focus-visible/active).
- Проектировать empty/loading/error состояния для каждого инструмента.
- Держать иерархию: один primary call-to-action на экран (кнопка броска).
- Резервировать монохром для содержания, цвет — для действия и результата.

## Rules: Don't

- Не использовать градиенты, неон, drop-shadow по умолчанию.
- Не использовать иконы-эмодзи в продакшен-UI (только в README/документации, если уж очень).
- Не плодить размеры — придерживаться шкалы 12/14/16/18/24/32/40.
- Не вводить новые цвета без обновления этого файла.
- Не использовать ambient motion (плавающие частицы, parallax) — это инструмент мастера, не лендинг.

## Anti-patterns

- Кнопка «Бросить» в виде дайс-3D-модели с физикой — нарушает спокойствие инструмента, замедляет работу за столом.
- Toast-уведомления о каждом броске — броски и есть основное действие, не нужно их сопровождать сторонними сообщениями.
- Sidebar-навигация с десятками систем сразу — пока систем мало, главная страница с карточками лучше.

## QA checklist

- [ ] Каждый интерактивный элемент имеет focus-visible.
- [ ] Контраст текста ≥ 4.5:1, крупного текста ≥ 3:1.
- [ ] Touch-target ≥ 44×44px у всех кнопок.
- [ ] Заголовки следуют иерархии H1 → H2 → H3 без пропусков.
- [ ] Нет сырых hex, всё из токенов.
- [ ] Все строки UI на русском, без англицизмов где есть русский эквивалент.
- [ ] Тестировано в Chrome и Safari на ноутбуке и iPhone-ширине.
